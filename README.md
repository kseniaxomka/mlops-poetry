Описание проекта
Этот проект демонстрирует использование Poetry — современного инструмента для управления зависимостями и виртуальными окружениями в Python.

В рамках практической работы мы:
1)Инициализировали Poetry-проект
2)Запустили Python-скрипт, который генерирует график на основе данных о температуре

Результат работы
После выполнения всех шагов программа создаёт файл temperature_24h.csv.

pyproject.toml — конфигурационный файл Poetry. Содержит в себе:
1)Название, версию, описание, автора
2)Список зависимостей с версиями
Выступает как более современный формат requirements.txt и setup.py

poetry.lock — файл фиксации версий. Это снимок точных версий всех зависимостей проекта + их хэши и маркеры окружения. Нужен для:
1)Повторяемых установок: у всех разработчиков в CI ставятся те же самые версии
2)Безопасности: хэши защищают от подмены пакетов
3)Скорости: не надо заново решать граф зависимостей — берутся уже зафиксированные
